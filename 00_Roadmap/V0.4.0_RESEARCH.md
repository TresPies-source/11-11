# UI Research Findings for v0.4.0

## App 1: Cursor IDE

### Key Observations

**Agent Representation:**
- **Agent Panel (Right Side):** Dedicated panel showing "IN PROGRESS" and "READY FOR REVIEW" tasks
- **Task Cards:** Each task shows:
  - Title
  - Status (Generating, Ready for Review)
  - Time estimate (30m, 45m)
  - Code changes (+37 -0, +135 -21)
  - Visual indicator (sparkle icon for generating)
- **Agent Chat:** Inline chat interface with agent responses
- **Model Selector:** Dropdown to choose between GPT-5, Claude Sonnet 4.5, Claude Opus 4.5, Gemini 3 Pro, Grok Code

**Layout:**
- **Three-Column Layout:**
  - Left: File tree / navigation
  - Center: Code editor (primary workspace)
  - Right: Agent panel (collapsible)
- **Flexible:** Agent panel can be shown/hidden
- **Context-Aware:** Agent panel shows relevant tasks for current workspace

**Onboarding:**
- Not visible on features page (likely on first launch)
- Focus on "Download" CTA

**Key Strengths:**
- Clear visual hierarchy (tasks organized by status)
- Real-time progress indicators ("Generating...")
- Code change metrics visible at a glance
- Model selection is prominent and easy to access
- Agent panel doesn't interfere with main workspace

**Key Weaknesses:**
- Agent panel is isolated (doesn't show in-editor suggestions)
- No visual representation of "which agent is active"
- Task list could become overwhelming with many tasks

## App 2: Windsurf Cascade

### Key Observations from Screenshot

**Agent Representation:**
- **Right-Side Chat Panel:** Dedicated "Cascade" panel for AI interaction
- **Conversational Interface:** Natural language chat with AI agent
- **Progress Indicators:** Shows what the agent is analyzing (e.g., "Analyzed app.py [1-100]", "Analyzed requirements.txt [1-11]")
- **Error Handling:** Red error messages inline (e.g., "Error: failed to run command pip: exec: 'pip': executable file not found in $PATH")
- **Action Buttons:** "Run this project" button at top of chat
- **Model Indicator:** Shows "Cascade 3.5 Sonnet" at bottom of chat panel

**Layout:**
- **Two-Column Layout:**
  - Left: Code editor (primary workspace)
  - Right: Cascade chat panel (persistent)
- **Tab System:** Multiple tabs at top (Explore, Refactor, Add Docstring)
- **Breadcrumb Navigation:** Shows file path (app.py > game_state > Explore)

**Agent Behavior:**
- **Proactive Analysis:** Agent automatically analyzes files when asked to help
- **Step-by-Step Progress:** Shows each file analyzed with line counts
- **Contextual Understanding:** Agent understands the project structure (Flask app with Python and JavaScript)
- **Error Detection:** Identifies missing dependencies and suggests fixes

**Key Strengths:**
- Persistent chat panel (doesn't hide)
- Clear progress indicators (shows what agent is doing)
- Inline error messages (easy to spot problems)
- Contextual awareness (agent knows the project structure)

**Key Weaknesses:**
- No visual indicator of "agent status" (is it thinking? waiting? done?)
- Chat panel takes up significant horizontal space
- No way to see multiple agent conversations simultaneously

## App 3: Replit Agent

### Key Observations from Screenshot

**Agent Representation:**
- **Bottom Status Bar:** "The agent was paused." message in purple bar at bottom
- **Agent Tab:** Dedicated "Agent" tab at top (alongside "features.tsx" tab)
- **Progress Tab:** Separate "Progress" tab showing agent activity
- **Visual Overlay:** Large modal/overlay showing "Transform Your Workflow" (likely agent-generated preview)

**Layout:**
- **Three-Column Layout:**
  - Left: File tree / navigation
  - Center: Code editor OR agent preview
  - Right: (not visible in this screenshot, but likely exists)
- **Tab System:** Multiple tabs for files and agent views
- **Bottom Status Bar:** Shows agent status persistently

**Agent Behavior:**
- **Pausable:** Agent can be paused/resumed by user
- **Preview Generation:** Agent generates visual previews of what it's building
- **Progress Tracking:** Separate "Progress" tab for detailed agent activity

**Key Strengths:**
- Agent status always visible (bottom bar)
- Agent can be paused (user control)
- Visual previews of agent work (not just code)
- Separate progress view (detailed activity log)

**Key Weaknesses:**
- Agent preview takes over entire center panel (disruptive)
- No indication of which agent is active (only one agent visible)
- Status bar message is passive (doesn't show active work)

## Key Insights for 11-11 v0.4.0

### What Works Well (Adopt)
1. **Persistent Agent Panel** (Cursor, Windsurf) - Don't hide agent activity
2. **Progress Indicators** (Windsurf) - Show what agent is analyzing/doing
3. **Status Bar** (Replit) - Always-visible agent status at bottom
4. **Task Organization** (Cursor) - Group agent work by status (IN PROGRESS, READY FOR REVIEW)
5. **Collapsible Panels** (Cursor) - User can hide agent panel when not needed

### What Doesn't Work (Avoid)
1. **Disruptive Overlays** (Replit) - Agent previews shouldn't take over entire workspace
2. **Isolated Agent Views** (11-11 current) - Agent activity should be visible from Workbench
3. **No Status Indicator** (Windsurf) - User should always know if agent is working
4. **Single Agent View** (all three) - 11-11 has multiple agents, should show them all

### Unique Opportunities for 11-11
1. **Multi-Agent Visualization:** Show Supervisor, Dojo, Librarian, Debugger simultaneously
2. **Agent Handoff Visualization:** Show when Supervisor routes to Librarian
3. **Seed Integration:** Show relevant seeds in agent panel (unique to 11-11)
4. **DojoPacket Preview:** Show current session as DojoPacket in agent panel
5. **Cost Tracking:** Show real-time cost of agent operations (Cost Guard integration)

## App 4: Linear

### Key Observations from Screenshot

**Sidebar Structure:**
- **Three-Tier Hierarchy:**
  - Top: Workspace selector ("Test Blog Workspace")
  - Middle: Core navigation (Inbox, My issues, Projects, Views, More)
  - Bottom: Team-specific navigation ("Test Blog Workspace" team with Issues, Projects, Views)
- **Collapsible Sections:** "Workspace", "Your teams", "Try" sections can expand/collapse
- **Action Items:** "Import issues", "Invite people", "Link GitHub" at bottom

**Main Content Area:**
- **Breadcrumb Navigation:** "Views > List" with star icon (favorite)
- **Filter Bar:** Persistent filter option at top
- **List/Board Toggle:** Switch between list and board views
- **Display Settings:** Right-side panel with extensive customization (grouping, ordering, visibility, properties)
- **Issue List:** Clean, minimal design with issue ID, title, date, assignee avatar

**Display Settings Panel:**
- **View Options:** List vs Board
- **Sub-grouping:** Organize issues by status, owner, priority
- **Ordering:** Sort by recency, priority, etc.
- **Visibility:** Show/hide completed issues, sub-issues
- **Properties:** Customize which columns to display (Priority, ID, Status, Labels, Project, Milestone, Due date, Created, Updated, Links, Assignee)

**Key Strengths:**
- Clean, minimal design (no visual clutter)
- Highly customizable views (user can configure everything)
- Persistent sidebar (always visible)
- Hierarchical navigation (workspace > team > view)
- Extensive filtering and grouping options

**Key Weaknesses:**
- Settings panel takes up significant space when open
- No visual preview of issues (text-only)
- No inline agent activity (all work is manual)

## Research Complete: Key Takeaways

### What 11-11 Should Adopt
1. **Persistent Sidebar** (Linear) - Navigation + projects + conversations
2. **Collapsible Agent Panel** (Cursor) - Right-side panel for agent activity
3. **Progress Indicators** (Windsurf) - Show what agent is analyzing/doing
4. **Status Bar** (Replit) - Always-visible agent status at bottom
5. **Customizable Views** (Linear) - User can configure layout and properties

### What 11-11 Should Avoid
1. **Disruptive Overlays** (Replit) - Agent previews shouldn't take over workspace
2. **Isolated Pages** (11-11 current) - Agent activity should be visible from Workbench
3. **Fixed Layouts** (Windsurf) - User should be able to customize panel sizes
4. **Single Agent View** (all three) - 11-11 has multiple agents, show them all

### Unique Opportunities for 11-11
1. **Multi-Agent Dashboard** - Show all agents (Supervisor, Dojo, Librarian, Debugger) in one view
